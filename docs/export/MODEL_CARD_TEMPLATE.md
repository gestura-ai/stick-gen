## üìù **What is a Model Card?**

A model card is a documentation file that provides essential information about a machine learning model. It's the standard way to share models on Hugging Face Hub and other platforms.

**Purpose**:
- Helps users understand what the model does
- Documents training data and methodology
- Explains intended use cases and limitations
- Provides usage examples
- Enables discoverability and reproducibility

**Industry Standard**: Used by OpenAI, Google, Meta, Anthropic, Stability AI, and all major AI labs.

- BibTeX citation
- License information
- Acknowledgments

### **Usage Examples to Include**

**Basic Generation**:
```python
from stick_gen import StickFigureGenerator

generator = StickFigureGenerator.from_pretrained("gestura-ai/stick-gen-v1")
animation = generator.generate("A person walking forward")
animation.save("output.mp4")
```

**Action-Conditioned Generation**:
```python
from stick_gen import ActionType

actions = [ActionType.WALK] * 125 + [ActionType.RUN] * 125
animation = generator.generate_with_actions(
    "A person walking then running",
    actions=actions
)
```

**Batch Generation**:
```python
prompts = [
    "A person jumping",
    "A person waving hello",
    "A person dancing"
]
animations = generator.generate_batch(prompts)
```

### **Limitations to Document**

**Technical Limitations**:
- 2D projection only (no depth)
- Fixed 10-second duration
- Stick figure representation (5 lines)
- Requires text embedding model

**Performance Limitations**:
- CPU inference: ~500ms per sequence
- GPU inference: ~50ms per sequence
- Memory: ~200MB model size

**Quality Limitations**:
- May struggle with complex multi-person interactions
- Limited fine-grained hand/finger movements
- Occasional temporal inconsistencies in rapid transitions
